## 📌 核心模块关系说明

### 1. 基础模块群

#### 1.1 用户管理 → 部门管理 → 角色权限

| 关系 | 说明 |
|------|------|
| 用户 → 部门 | 每个用户属于一个部门（department_id） |
| 用户 → 角色 | 每个用户分配一个角色（role_id） |
| 角色 → 权限 | 角色关联模块权限（permissions JSON） |
| 部门 → 团队 | 部门数据供团队管理调用 |

**数据流**：
```
用户管理（创建/编辑用户）
    ↓ 选择部门
部门管理（维护部门数据）
    ↓ 读取部门
团队管理（邀请成员时选择部门）
```

#### 1.2 角色权限说明

权限树结构（10个模块）：
- 仪表盘(dashboard): 查看
- 项目管理(projects): 查看、创建、编辑、删除
- 任务管理(tasks): 查看、创建、编辑、删除
- 团队管理(team): 查看、创建、编辑、删除
- 报表(reports): 查看、导出
- 文档管理(documents): 查看、创建、编辑、删除、导出
- 资源管理(resources): 查看、创建、编辑、删除
- 问题管理(issues): 查看、创建、编辑、删除
- 系统设置(settings): 查看、编辑
- 用户管理(users): 查看、创建、编辑、删除

---

### 2. 项目管理群

#### 2.1 项目 → 任务 → 子任务

| 关系 | 说明 |
|------|------|
| 项目 → 任务 | 一个项目包含多个任务（1:N） |
| 任务 → 子任务 | 任务可以包含子任务（1:N） |
| 任务 → 负责人 | 任务分配给团队成员（user_id） |
| 任务 → 项目 | 任务归属于项目（project_id） |

#### 2.2 任务的多视图展示

同一任务数据，可以通过三种视图查看：

| 视图 | 用途 | 场景 |
|------|------|------|
| **甘特图** | 时间线视图 | 展示任务时间安排、依赖关系 |
| **看板视图** | 状态流转 | 展示任务在不同状态间流转 |
| **日历视图** | 日历展示 | 按日期查看任务安排 |

## 📝 总结

1. **核心数据**：用户 → 角色 → 权限，是整个系统的基础
2. **项目管理**：以项目为根，任务为核心，形成树状结构
3. **协作支持**：团队、评论、审批流为项目执行提供支撑
4. **辅助功能**：统计、文档、资源、AI助手提供增值服务
5. **安全保障**：审计日志、权限控制、备份恢复保障系统安全
