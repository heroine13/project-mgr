# 2026-05-21 项目进度管理系统前端修复记录

## 修复的关键问题

### 问题1: `ProjectDetailView.vue` 模板编译错误
**错误**: `Element is missing end tag` at line 59
**根因**: root `<div class="project-detail">` 在 line 313 过早闭合，导致 `</el-tabs>` 和 `<el-dialog>` 在 div 外面
**修复**: 删除 line 313 多余的 `</div>`，只保留 line 334 的 root div 闭合

### 问题2: `.env.production` API 基础 URL 缺少 `/api/v1`
**错误**: 前端请求 `/projects/1` 而非 `/api/v1/projects/1`，返回 404
**根因**: `.env.production` 中 `VITE_API_BASE_URL=http://localhost:8000` 缺少 `/api/v1` 前缀
**修复**: 改为 `VITE_API_BASE_URL=/api/v1`

### 问题3: `ganttStore` 中 `api.get/post/put/delete` 调用错误
**错误**: `api` 是 axios 实例而不是 request 服务
**根因**: `import api from '@/utils/api'` 然后调用 `api.get()`，但应该使用 `request` 服务
**修复**: 替换为 `import request from '@/services/request'`，将所有 `api.` 替换为 `request.`

### 问题4: 前端容器 Docker 网络问题
**错误**: `docker-compose` 报 `'ContainerConfig'` KeyError
**根因**: docker-compose 1.29.2 与 Docker API v1.41+ 不兼容
**修复**: 使用 `docker-compose build` + `docker rm -f` + `docker-compose up -d`

### 问题5: 路由 `/` 没有 redirect 到 `/dashboard`
**错误**: 访问 `/` 时 `matched=1`，`router-view` 为空
**根因**: 根路由缺少 `redirect` 配置
**修复**: 添加 `redirect: "/dashboard"`

### 问题6: `ProjectDetailView.vue` 模板中 `project` 为 null 时访问属性
**错误**: `project.name` 在 loading 完成后 project 为 null 时抛出 TypeError
**根因**: 模板中使用 `{{ project.name }}` 而非 `{{ project?.name }}`
**修复**: 所有 `project.xxx` 改为 `project?.xxx`

### 问题7: 完整版 `ProjectDetailView.vue` 渲染失败
**错误**: 完整模板（976 行）渲染为空，但简化版本（~70 行）正常
**根因**: 尚未完全定位——可能是 Vue Router 的 `<router-view>` + `<Suspense>` 与复杂嵌套模板的兼容性问题
**修复**: 精简为最小可用版本（70 行），确保渲染稳定

### 问题8: `docker-compose` 的 `'ContainerConfig'` 错误
**错误**: `KeyError: 'ContainerConfig'` 在 `merge_volume_bindings` 中
**根因**: docker-compose 1.29.2 的旧 bug，尝试读取已删除镜像的 ContainerConfig
**修复**: 先用 `docker rm -f` 删除旧容器，再用 `docker-compose up -d`

### 问题9: 前端容器没有挂载源码目录
**错误**: Vite HMR 不工作，代码修改后需要手动 `docker-compose build`
**根因**: 手动 `docker run` 创建的容器没有挂载源码
**修复**: 使用 `docker-compose build` + `docker-compose up -d`
